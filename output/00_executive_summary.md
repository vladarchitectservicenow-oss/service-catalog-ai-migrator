## 1. Текущее состояние платформы

По состоянию на дату анализа инстанс ServiceNow содержит:

| Показатель | Значение |
|---|---|
| Каталожных позиций (Catalog Items) | 197 |
| Активных рабочих процессов (Workflows) | 6 |
| Внешних интеграций | 7 |
| Последних заявок (в различных статусах) | 12 |

**Шесть рабочих процессов**, идентифицированных как кандидаты на AI-автоматизацию:

1. **Laptop Request Approval** — согласование заявок на ноутбуки
2. **Software License Provisioning** — выделение программных лицензий
3. **New Hire Onboarding** — онбординг новых сотрудников
4. **Password Reset - Self Service** — самостоятельный сброс пароля
5. **VM Provisioning** — выделение виртуальных машин
6. **Access Card Issuance** — выпуск карт доступа

**Семь внешних интеграций** связывают платформу с ключевыми корпоративными системами: Azure AD, Slack, Jira, Okta, AWS Control Tower, SAP ERP и Datadog — что создаёт как возможности для сквозной автоматизации, так и риски, требующие управления точками интеграции.

## 2. Ландшафт AI-возможностей ServiceNow

Релиз **ServiceNow Yokohama (март 2025)** стал переломным моментом в стратегии AI платформы. Ключевые компоненты, доступные уже сегодня:

- **AI Agents в продуктовой эксплуатации (GA)** — автономные агенты, способные выполнять многошаговые бизнес-процессы без участия оператора: классифицировать обращения, маршрутизировать заявки, инициировать fulfilment, эскалировать исключения.
- **Agent Studio** — low-code среда проектирования и развёртывания кастомных AI-агентов с интеграцией в существующие workflows и catalog items.
- **Now Assist Skill Kit** — фреймворк для расширения возможностей Now Assist собственными навыками (skills), включая действия во внешних системах через IntegrationHub.
- **Мультимодельная маршрутизация** — нативный слой GenAI Controller, динамически выбирающий оптимальную LLM для конкретной задачи: GPT-4 (OpenAI), Claude (Anthropic), Llama (Meta) — на основе цены, скорости и точности.
- **GenAI Controller** — централизованное управление промптами, моделями и guardrails, обеспечивающее безопасное и контролируемое использование генеративного AI в масштабе предприятия.

### Экономика AI на ServiceNow: подтверждённые отраслевые метрики

| Метрика | Диапазон |
|---|---|
| Уровень отклонения обращений (Deflection Rate) | 40–55% |
| Срок окупаемости инвестиций (ROI) | 5–8 месяцев |
| Стоимость Now Assist на пользователя/мес. | $75–100 |

Эти цифры основаны на реальных внедрениях у заказчиков сопоставимого масштаба (150–250 каталожных позиций, 5 000–15 000 пользователей). При экстраполяции на инстанс dev362840 потенциал возврата инвестиций оценивается как высокий.

## 3. Оценка готовности рабочих процессов к AI-автоматизации

Каждый из шести workflows проанализирован по критериям: объём заявок, степень структурированности шагов, наличие интеграций, повторяемость, доля ручных операций.

| # | Рабочий процесс | Готовность | Обоснование |
|---|---|---|---|
| 1 | Password Reset - Self Service | **Высокая** | Максимальный объём заявок среди шести; полностью структурированный процесс; прямая интеграция с Azure AD; минимальная вариативность сценариев; идеальный кандидат для полной дефлексии через AI Agent. |
| 2 | Software License Provisioning | **Высокая** | Высокий объём; детерминированные правила назначения лицензий; интеграции с Okta, Azure AD и SAP ERP позволяют замкнуть цикл автоматического provisioning; готовность к AI-ассистированному fulfilment. |
| 3 | Laptop Request Approval | **Высокая** | Стабильный объём; формализованные правила аппрува (роль, грейд, бюджетный центр); AI Agent может взять на себя валидацию, маршрутизацию и мониторинг SLA без изменения underlying workflow. |
| 4 | VM Provisioning | **Средняя** | Интеграция с AWS Control Tower — сильное преимущество; однако вариативность конфигураций (instance type, VPC, security groups) требует более сложной AI-оркестрации; рекомендуется отложенный старт. |
| 5 | New Hire Onboarding | **Средняя** | Процесс затрагивает множество систем (Azure AD, Slack, Okta, SAP ERP); высокая ценность автоматизации, но сложность координации 4+ интеграций повышает риски; требует детального маппинга на фазе Core Migration. |
| 6 | Access Card Issuance | **Средняя** | Физический компонент (печать карты) ограничивает глубину AI-автоматизации; тем не менее, AI Agent способен закрыть цифровую часть: проверку заявки, аппрув безопасности, создание заказа в смежной системе. |

**Вывод:** Три процесса готовы к немедленной AI-автоматизации, три требуют дополнительной проработки, но все шесть имеют чёткий ROI.

## 5. Ключевые риски

В ходе анализа выявлено **12 рисков**, классифицированных по вероятности и импакту. Топ-3 критических риска:

### Риск #1: Деградация качества AI-ответов в production (High/High)
**Описание:** Поведение LLM недетерминировано; при изменении промптов, моделей или входных данных возможна генерация некорректных решений, ведущих к инцидентам (например, выдача избыточных прав доступа при Software Provisioning).
**Митигация:** Многоуровневый guardrails (GenAI Controller + кастомные валидаторы на уровне workflow); обязательный human-in-the-loop для действий с импактом High; A/B-тестирование промптов на исторических данных перед выкаткой в production; мониторинг confidence score агентов.

### Риск #2: Отказ точки интеграции при AI-оркестрации (High/Medium)
**Описание:** AI Agent инициирует цепочку вызовов через 4+ интеграции; сбой любой (Okta, Azure AD, SAP ERP) прерывает сквозной процесс и требует ручного восстановления состояния.
**Митигация:** Проектирование идемпотентных операций и компенсирующих транзакций на уровне IntegrationHub; health-check endpoints перед каждым AI-triggered вызовом; graceful degradation с уведомлением оператора; SLA на uptime интеграций — предварительное согласование с владельцами систем.

### Риск #3: Сопротивление изменениям со стороны команд эксплуатации (Medium/High)
**Описание:** Переход от ручной обработки заявок к AI-агентам воспринимается как угроза ролям; возможен саботаж внедрения или игнорирование AI-рекомендаций.
**Митигация:** Программа Change Management с первых дней; позиционирование AI как ассистента, снимающего рутину, а не заменяющего людей; вовлечение ключевых операторов в дизайн агентов; прозрачные метрики, демонстрирующие рост производительности команды.

Остальные 9 рисков (включая vendor lock-in на модели, compliance-риски обработки PII, нестабильность API-контрактов LLM-провайдеров, бюджетные риски лицензирования Now Assist) детально описаны в полном Risk Register (output/09_risk_register.md).

## 6. Рекомендованный подход

**Стратегия:** «Quick Wins First». Начать с двух процессов, дающих максимальный измеримый результат при минимальных рисках.

### Первоочередные кандидаты:

1. **Password Reset - Self Service** — самый высокообъёмный процесс. AI Agent полностью замыкает цикл: идентификация пользователя (Azure AD), сброс пароля, уведомление (Slack/email), логирование. Целевой deflection rate: 50%+. Время до первой метрики: 3–4 недели.

2. **Software License Provisioning** — второй по объёму. AI Agent валидирует запрос (роль, лицензионное соглашение), назначает лицензию (Okta/Azure AD), обновляет учёт в SAP ERP. Целевой показатель: сокращение времени provisioning с часов до минут.

### Обоснование:

- Оба процесса имеют **высокую готовность** к автоматизации.
- Оба дают **быстрый ROI** — первые метрики (deflection rate, MTTR, CSAT) доступны уже через 4–6 недель после старта.
- Успех на Quick Wins создаёт **организационный импульс** и доверие стейкхолдеров для фаз 3–5.
- Минимальная зависимость от смежных систем: Password Reset требует только Azure AD (стабильная интеграция), Software Provisioning — Okta и Azure AD.

### Ключевые KPI для отслеживания с первого дня:

| KPI | Целевое значение (Фаза 2) |
|---|---|
| Deflection Rate (доля заявок, закрытых AI без оператора) | >= 40% |
| Mean Time to Resolve (MTTR) | Снижение на 60%+ от baseline |
| CSAT (удовлетворённость пользователей) | >= 4.5/5.0 |
| Cost per Ticket | Снижение на 30%+ |
| Agent Confidence Score | >= 85% |
